Actúa como arquitecto principal de un software POS comercial de nivel enterprise.

Contexto del proyecto:
- Tipo: Software POS modular
- Modelo: SaaS multi-cliente
- Sectores: Restaurantes, bares y retail
- Enfoque: Control operativo, antifraude, escalabilidad
- Arquitectura: Backend centralizado + POS cliente (web / app)
- Operación: Online con soporte offline

Objetivo:
Generar el DOCUMENTO TÉCNICO COMPLETO del sistema, coherente entre todos los módulos,
sin contradicciones internas y con nivel suficiente para que un equipo de desarrollo
pueda implementarlo sin tomar decisiones ambiguas.

3. Módulos del sistema

   - Pantallas exactas
   - Flujos principales

   - Reglas de negocio innegociables
   - Casos límite reales (fraude interno incluido)
   - Flujo offline (si aplica)
   - Eventos de auditoría

   Módulos obligatorios:
   - Autenticación y sesiones
   - Usuarios y roles
   - Turnos
   - Caja y pagos
   - Ventas (TPV / POS)

   - Inventario
   - Mesas y salón
   - Clientes y fidelización
   - Cocina y producción
   - Reportes y analítica
   - Auditoría y seguridad
   - Delivery e integraciones

4. Modelo de datos
   - Tablas principales
   - Relaciones
   - Restricciones críticas
   - Campos obligatorios

5. API

   - Métodos

6. Flujo offline y sincronización

7. Seguridad y antifraude

8. Auditoría

9. Escalabilidad y multi-sucursal
